Drop duplicates: because the merging is a 'one against multiple addresses' operation.¶

  • In the original file, each address is paired with multiple dentists and multiple specialties.
  • In the file produced after the FTE calculation, each address has only one entry for each specialty, because the head values have been summed.
  • When these two files merge by addresses so that the FTE file can regain the X, Y and other columns, you will get as many rows as the origianl file has. We are using the FTE file as our anchor, thus, we need to drop the duplicates.

Make sure that you perform this step when the Medicaid, CHIP and BOD-stand-alone parts are still separate! Once the three parts are combined into a final file, you shouldn't be dropping duplicates because there are providers who accept both Medicaid and CHIP and will appear twice.¶
